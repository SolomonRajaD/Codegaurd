This project demonstrates a comprehensive security audit of a conceptual FinTech application. Below is a sample excerpt from a vulnerability report, highlighting a critical finding.

Vulnerability Report: FT-API-001
[CRITICAL] SQL Injection in /api/v1/transactions

**Description:** The `transactionId` parameter in the `/api/v1/transactions` endpoint is vulnerable to a time-based blind SQL injection attack. An attacker can extract sensitive database information by crafting malicious payloads.

**Proof of Concept:**
`GET /api/v1/transactions?transactionId=1' AND (SELECT 1 FROM (SELECT(SLEEP(5)))a)--`

**Impact:** An unauthorized attacker could potentially exfiltrate the entire user database, including usernames, hashed passwords, and transaction histories.

**Recommendation:** Immediately implement parameterized queries (prepared statements) for all database interactions to prevent SQL injection vulnerabilities.
